Generic design of data sharing: - Primary re-use of data (during health provisioning) is mediated via Data Station that contains Shared Health Record for a given group of patients. Data Stations are envisaged at county level. This intermediate solution of a 'persistent copy' of all data is necessary for as long as Point of Service apps aren't sufficiently robust to support a fully federated data sharing architecture. - All Point of Service apps can publish (PUT) data into the Data Station. This could be a separate FHIR Resource (e.g. MedicationDispense by Pharmacy, or a QuestionnaireResponse by the Community Health Worker) or a full International Patient Summary (IPS) - All Point of Service apps can read (GET) data from the Data Station. By default this is the IPS, but can also be a sub-set, depending on data governance rules
